# Trading Assistant

Trading Assistant — Telegram-бот для объяснимого анализа криптофьючерсного
рынка. Он получает публичные данные Bitunix, сканирует доступные инструменты,
показывает графики и рыночный контекст, а также отправляет настраиваемые
уведомления о Fair Value Gap (FVG).

Бот не открывает сделки и не является финансовой рекомендацией. Торговые
сигналы допускаются к автоматической рассылке только после проверки стратегии
на исторических данных в Strategy Lab.

## Возможности

- Scanner: поиск и ранжирование наиболее интересных фьючерсных инструментов;
- объяснимый контекст: EMA, объём, funding, market structure, Fibonacci и FVG;
- графики BTC и выбранных пар прямо в Telegram;
- уведомления о предварительных и подтверждённых 15-минутных FVG через общий
  WebSocket Bitunix;
- фильтры по инструментам, цене и размеру FVG;
- Strategy Lab: бэктесты, ежемесячные отчёты и walk-forward-проверки;
- Decision Gate: защита от автоматической отправки непроверенных сетапов.

## Запуск

1. Создайте `.env` рядом с `bot.py`, используя `.env.example`:

   ```env
   TELEGRAM_TOKEN=токен_от_BotFather
   ```

2. Установите зависимости в активированном виртуальном окружении:

   ```bash
   .venv/bin/python -m pip install -r requirements.txt
   ```

3. Запустите бота:

   ```bash
   .venv/bin/python bot.py
   ```

   Сейчас бот работает в публичном режиме: проверка Telegram ID и панель
   одобрения доступа отключены. Чтобы вернуть закрытый режим, переключите
   `PUBLIC_ACCESS_ENABLED` в `handlers/auth.py` на `False`.

## Команды

- `/start` — статус системы и список команд.
- `/myid` — ваш Telegram ID для настройки доступа.
- `/btc` — цена, диапазон и объём BTCUSDT.
- `/chart BTCUSDT 1h` — график с EMA, текущей ценой и открытыми FVG;
  таймфреймы: `15m`, `1h`, `4h`.
- `/scan` — TOP‑5 кандидатов Scanner с рыночной структурой, Confluence,
  FVG/Fibonacci‑контекстом и funding.
- `/status` — состояние Strategy Lab, Decision Gate и авторассылки.
- `/fvg_alert on|off` — включить/выключить модуль 15m FVG.
- `/fvg_pre_alert on|off` — включить/выключить предварительный FVG в точке T−3.
- `/fvg_symbol add ETHUSDT` — добавить доступный фьючерсный инструмент Bitunix.
- `/fvg_symbol remove ETHUSDT` — удалить инструмент из наблюдения.
- `/fvg_price BTCUSDT 50000 90000 both` — включить фильтр цены сигнала;
  вместо открытой границы используйте `-`, режимы: `pre`, `confirmed`, `both`.
- `/fvg_price BTCUSDT off` — отключить ценовой фильтр инструмента.
- `/fvg_size` — открыть шаблон фильтра размера зоны в процентах или долларах.
- `/fvg_stats` — количество обнаруженных FVG и отправленных уведомлений.

FVG-уведомления используют публичные рыночные данные Bitunix и не требуют
API-ключа. Основной источник — один общий WebSocket для активных инструментов;
REST используется для начального заполнения и восстановления пропусков. В
уведомлениях FVG нет фильтра по объёму. Подробности: [docs/FVG_ALERTS.md](docs/FVG_ALERTS.md).

`/scan` показывает аналитические кандидаты. Итоговая торговая рекомендация
появится только после успешной валидации стратегии в Strategy Lab.

## Автоматические сетапы

По умолчанию автоматическая рассылка выключена. После того как Strategy Lab
одобрит стратегию, добавьте в `.env`:

```bash
AUTO_ALERTS_ENABLED=true
AUTO_ALERTS_INTERVAL_MINUTES=15
```

Бот будет сканировать рынок по расписанию и отправит сообщение с графиком
только когда кандидат прошёл Strategy Validation, Probability, Confidence и
Decision Gate (включая Confluence не ниже 65). Пока в отчёте указано `approved: false`, сообщений-сетапов не
будет: это защищает от неподтверждённых рекомендаций.

## Проверки

```bash
.venv/bin/python -m unittest discover -s tests -v
```

Для честной проверки на последовательных out-of-sample периодах используйте
walk-forward режим (90 дней контекста, 30 дней теста по умолчанию):

```bash
.venv/bin/python run_backtest.py --data-file data/historical/btcusdt_15m.csv --walk-forward
```

Для лёгкого исследования по отдельным месяцам (с общей годовой сводкой после
всех месяцев) используйте:

```bash
.venv/bin/python run_backtest.py --data-file data/historical/btcusdt_15m.csv --monthly
```

Месячный режим не включает автосетапы: для этого по-прежнему нужен отдельный
walk-forward отчёт.

Чтобы запускать месяц за месяцем и не нагружать компьютер, выберите один
календарный месяц:

```bash
.venv/bin/python run_backtest.py --data-file data/historical/btcusdt_15m.csv --month 2026-03 --report-file data/reports/btcusdt_2026-03.json
```

После сохранения отдельных месяцев объедините их без повторного бэктеста:

```bash
.venv/bin/python aggregate_monthly_reports.py data/reports/btcusdt_2026-*.json
```

Бэктест по умолчанию учитывает 6 bps комиссии и 2 bps неблагоприятного
проскальзывания на каждую сторону сделки. Их можно изменить параметрами
`--fee-bps-per-side` и `--slippage-bps-per-side`.

Текущий зафиксированный риск-параметр стратегии: stop-loss = `1R`, take-profit
= `2R` (соотношение риск/прибыль 1:2). При будущей переработке входов этот
параметр сохраняется, пока не будет отдельно изменён.

FVG-входы новой стратегии формируются только на `15m`. Часовые и
четырёхчасовые FVG не используются для открытия позиций.
